# Vancouver End-to-End Housing Insights

## Introduction
This repository contains an end-to-end data pipeline and machine learning project focused on Vancouver's housing market. The goal of this project is to provide data-driven insights into housing properties, local areas, crime rates, and business densities in Vancouver, culminating in a machine learning model to predict land values and cluster neighborhoods. The project encompasses the complete data lifecycle: data ingestion, transformation, exploratory data analysis (EDA), machine learning modeling, and reporting.

## Project Structure

The repository is organized into several key directories and files:

### Files in the Root Directory
- **`main.py`**: The main entry point for executing the automated data pipeline. Running this file performs data ingestion and transformation sequentially.
- **`.gitignore`**: Specifies intentionally untracked files to ignore in Git.

### Folders and their purposes

#### `data/`
This folder stores the datasets used and generated by the pipeline.
- **`landing/`**: Stores raw data extracted directly from the sources during the ingestion step.
- **`staging/`**: Stores the cleaned and engineered data outputted during the transformation step, ready for EDA and modeling.

#### `src/`
The main source code directory, modularized by pipeline stages.

- **`ingestion/`**: Contains scripts to extract raw data.
  - `extract_business_licences.py`: Fetches business licence data.
  - `extract_crime.py`: Fetches crime incident data.
  - `extract_local_areas.py`: Fetches Vancouver local area boundaries.
  - `extract_property_tax_report.py`: Fetches property tax records.
  - `ingestion_utils.py`: Utility functions to assist with API calls and data fetching.

- **`transformation/`**: Contains scripts to clean and prepare the extracted data.
  - `transform_business_licences.py`: Cleans and filters business data.
  - `transform_crime.py`: Cleans and aggregates crime statistics.
  - `transform_local_areas.py`: Processes area boundaries.
  - `transform_property_tax.py`: Cleans housing/property tax and value data.
  - `transform_utils.py`: Helper functions for data cleaning and transformation.

- **`exploratory/`**: Contains Jupyter Notebooks used for Exploratory Data Analysis (EDA).
  - `EDA_business_licences.ipynb`, `EDA_crime.ipynb`, `EDA_local_areas.ipynb`, `EDA_property_tax.ipynb`, `EDA_external_features.ipynb`: Detailed analysis, visualizations, and profiling for each data domain.

- **`machine_learning/`**: Contains notebooks for modeling.
  - `linear_regression.ipynb`: Develops a Multiple Linear Regression model to predict current land values and utilizes K-Means clustering for grouping neighborhoods based on engineered features.

- **`report/`**: Contains reporting assets.
  - `generate_report.py`: Functions to compile results.
  - `report.ipynb`: A comprehensive notebook summarizing findings, visualizations, and business recommendations.

## How to Run the Data Pipeline

To execute the data pipeline and generate the ready-to-use CSV files, you only need to run the `main.py` script. 

The script will automatically perform two key steps:
1. **Data Ingestion**: It will automatically fetch data from the varied sources and save the raw `.csv` files into the `data/landing/` directory.
2. **Data Transformation**: It will then process, clean, and merge the raw data, saving the finalized `.csv` files into the `data/staging/` directory.

Run the following command from the root of the repository:

```bash
python main.py
```

Once the script completes successfully, you can find the cleaned datasets in the `data/staging/` directory, ready for the Jupyter notebooks in `src/exploratory/` and `src/machine_learning/`.
